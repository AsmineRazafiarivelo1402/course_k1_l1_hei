Configuration de Linux pour l'accès à distance

Pour que Screens puisse se reconnecter à votre PC Linux, il doit être configuré pour l'accès à distance. Dans notre exemple, nous montrons comment activer le Bureau à distance sur Ubuntu 22.04. Les autres distributions peuvent varier, veuillez donc vous reporter au guide de l'utilisateur de votre distribution pour obtenir des instructions.

Activer le bureau à distance

Selon votre distribution Linux, les détails de configuration peuvent varier, mais la plupart des environnements de bureau Linux modernes fournissent généralement une fonctionnalité appelée Partage de bureau ou Bureau à distance.

  1. Cliquez sur les icônes qui se trouvent à l'extrême droite de la barre de menu système et cliquez sur Paramètres.
  2. Dans la fenêtre Paramètres, sélectionnez Partage et cliquez sur le bouton de commutation supérieur pour activer le service.
  3. Cliquez sur Bureau à distance.
  4. Activez Bureau à distance (cliquez sur le commutateur).
  5. Sélectionnez Activer le protocole VNC hérité.
  6. Cliquez sur les 3 points et sélectionnez soit Les nouvelles connexions doivent demander l'accès, soit Exiger un mot de passe.
  7. Dans le champ Mot de passe, saisissez un mot de passe ou utilisez celui qui a été généré automatiquement. Si vous avez sélectionné Exiger un mot de passe, vous devrez utiliser ce mot de passe lorsque vous vous connecterez depuis Screens.
  8. Dans notre exemple, le nom d'hôte à utiliser lors de la création d'une connexion à votre ordinateur dans Screens serait Ubuntu-22-04.local (sans le préfixe vnc://).

Facultatif : Installer un serveur SSH

Installation d'OpenSSH

Pour installer OpenSSH, ouvrez un terminal et exécutez les commandes suivantes avec les autorisations de superutilisateur.

Pour les distributions utilisant le système de gestion de paquets apt comme Ubuntu, Debian ou Linux Mint

$ sudo apt-get install openssh-server openssh-client

Pour les distributions utilisant le système de gestion de packages rpm, telles que Red Hat Enterprise, CentOS ou Fedora

$ yum -y install openssh-server openssh-client

Tester la connexion

Avant de continuer, utilisez la commande Terminal suivante pour confirmer que votre serveur OpenSSH fonctionne correctement.

$ nc -v -z 127.0.0.1 22

Rendre votre ordinateur accessible à distance

Si vous souhaitez vous connecter à votre ordinateur à distance, nous vous recommandons d'utiliser un service tel que No-IP. Vous trouverez des instructions dans cet article.

Sélection d'un moniteur

Afin de sélectionner un moniteur particulier sur votre ordinateur, vous devez spécifier un numéro de port différent. Vous trouverez plus de détails dans cet article.

Configuration de Screens

Une fois que vous avez terminé, vous pouvez maintenant créer une nouvelle connexion afin de vous connecter à votre PC Linux.
